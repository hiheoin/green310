시맨틱태그

- html5의 가장 획기적인 태그
- semantic : '의미론적인'이라는 사전적의미를 갖는다.
- 시맨틱웹은 컴퓨터 프로그램이 코드를 읽고 의미를 인식할 수 있는 지능형 웹 언어를 표현한다.

특징
- 각각의 의미하는 바가 정해져 있으므로 위치가 적절치 않은 곳에 쓰일 경우 웹 접근성 부분에서 오류로 파악하기도 한다.
- 오래된 브라우저의 경우, 모든 시맨틱태그를 div로 대체한다.
- 기본 구조가 블럭박스로 인식된다.

시맨틱종류

  • header태그 - 머리말, 페이지 제목, 페이지 소개, 로고, 내비게이션(메뉴)를 담는다.
  • nav태그 - 링크가 포함된 메뉴리스트들의 내비게이션
  • section태그 - html문서에 포함된 독립적인 공간으로 나누고자 할 때 사용하며, 제목요소를 포함하고 있는 경우가 많다.
  • article태그 - 섹션태그와 비슷한 성질을 가지고 있는 소단락 구간, 블로그포스트/기사/논평 등으로 활용, 제목태그는 h2~h6을 담는다.
  • aside태그 - 본문 흐름에서 벗어나는 구조
  • footer태그 - 업체정보, 저작권정보, 사이트맵 등 사이트의 주요 알림 내용을 설정한다.

  • figure태그 - 삽화, 다이어그램 등 문서의 주요 흐름의 독립적인 콘텐츠를 정의할 때 사용
  • figcaption태그 - figure태그 내부에 첫번째 혹은 마지막 자식태그로 사용되면서, figure태그의 내용을 정의/설명할 때 사용
